1. **What did we do well?**  
    We were able to compile a very large number of resources, organizations, and events from various sources in a format that our users could navigate easily. We were also able to implement helpful features like filtering and sorting by location.

2. **What did we learn?**  
    We learned how to programmatically scrape data, load it into a database, and query it using an API we designed. We also learned how to dynamically render our content and query subsets of our data for filtering, sorting, and searching. In terms of soft skills, we learned how to work on a project with multiple moving parts. Communicating with each other, asking for help when needed, and ensuring the process moved smoothly was very important.

3. **What did we teach each other?**  
    We taught each other how to navigate challenges and work through technical difficulties. We also learned new skills from group members who had previously worked with some parts of the tech stack.

4. **What can we do better?**  
    We can implement extra features such as an interactive map and flesh out our events model more. We could also separate our instances into more specific models like medical aid, legal services, and other categories.

5. **What effect did the peer reviews have?**  
    Having an outsider perspective on our project was very helpful. Hearing what a potential user would actually like to use gave us good insight into what to implement. Reviewing our developer’s website also helped us anticipate possible problems in our own project.

6. **What puzzles us?**  
    We are puzzled by our struggle to get API keys from the 211 API portal and the difficulty we had in scraping events. We were able to circumvent these issues, but it was frustrating to have our progress hindered by factors outside our control.
